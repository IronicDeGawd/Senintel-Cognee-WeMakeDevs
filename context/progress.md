## Lessons

- Dynatrace MCP auth: the server accepts a **Platform Token** (`dt0s16.`), an OAuth client (`dt0s02.`), or **browser OAuth** (just `DT_ENVIRONMENT`, no token). A classic **API token** (`dt0c01.`) is NOT a JWT → fails `401 Could not parse JWT`. For a local probe, blank `DT_PLATFORM_TOKEN` and let the Authorization-Code-Flow open a browser. Headless poller needs a real `dt0s16` token (OAuth cache lasts only ~300s).
- Dynatrace MCP v1.8.7 `execute_dql` returns a markdown `{"text": ...}` summary (App-UI render), not raw `{"records": [...]}`. Code must `.get("records", [])` and degrade gracefully; don't assume structured records.
- `npx` on Windows is `npx.ps1` — `Start-Process npx` fails "not a valid Win32 application"; use `npx.cmd`. The ADK/mcp stdio client spawns it fine via the package name.
- **Never rewrite `.env` with PowerShell `Set-Content -Encoding utf8`** — PS 5.1 adds a UTF-8 BOM. The BOM glues onto line 1's key (`﻿GOOGLE_GENAI_USE_VERTEXAI`), so `os.environ.get` returns None and ADK silently falls back to GEMINI_API (no key) → `ValueError: No API key`. pydantic `settings` still reads fine (field default masks it), so only the ADK agent path breaks — confusing. Fix: write BOM-free via `[System.IO.File]::WriteAllText(path, text, (New-Object System.Text.UTF8Encoding($false)))`.
- ADK agents call Gemini via their OWN model client (`google.adk.models.google_llm`), NOT `shared.llm.generate*` — so the tenacity 429/5xx retry in `shared/llm.py` does NOT protect the agent/runner path. A multi-call agent turn (LLM→tool→LLM→tool→LLM-summary) can exhaust quota and raise `_ResourceExhaustedError` on a later call. To verify an agent build under tight quota, read the tool-call trace (`[tool call]`/`[tool result]`) — if the tools fired and returned ok, orchestration is proven even when the final narration call 429s.
