# Plans — WeMakeDevs extension

One file per feature: `plan/<feature>.md`. Write the spec BEFORE any
multi-file change (per workflow rules).

## Spec template
```
# <feature>

**Why:** the problem / motivation.
**Goal:** what "done" looks like, with the sim/real switch if applicable.

## Approach
- files to create/modify
- data flow
- build sequence (small, committable units)

## Verification
- how it's proven to work (tests, endpoint, UI)
```
