## Event
- **Organizer:** WeMakeDevs · **Partner:** Cognee
- **Dates:** June 29 – July 5, 2026. **Submissions close July 5, 2026 (TODAY).**
- **Theme:** "Build AI that doesn't forget" using Cognee's memory layer.
- **Team:** max 4; solo allowed. Submit via Google Form.

## Core requirement
- **Cognee MUST power the memory.** That is the only hard rule.
- Leverage the four lifecycle APIs:
  - `remember()` — ingest text/files/URLs into permanent memory.
  - `recall()` — query memory across sessions (auto-routed search).
  - `improve()` / memify — enhance memory over time.
  - `forget()` — surgically delete datasets.
- Prior work / existing projects allowed. AI assistants allowed.

## Judging (6 categories)
1. Potential Impact — real problem solved with persistent memory.

4. **Best Use of Cognee** — depth across the memory lifecycle + hybrid graph-vector layer.
5. User Experience — polish.
